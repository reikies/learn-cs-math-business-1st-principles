# Algebra — The Study of Structure

> Algebra asks: "Forget what the objects ARE. What can we DO with them?"
> It studies operations and their properties — the skeleton underneath all of mathematics.

**Prerequisites:** [Number Systems](number-systems.md), [Set Theory](../01-foundations/set-theory.md)
**What it enables:** Linear algebra, cryptography, coding theory, symmetry in physics, type systems.

---

## Why Algebra Exists

In number systems, we noticed patterns that REPEATED across different systems:
- Addition of integers and addition of rationals both follow the same rules.
- Modular arithmetic and matrix multiplication both have similar structure.
- Symmetries of a square and permutations of a list both obey the same laws.

Algebra says: **strip away the specific objects and study the structure itself.** If two systems follow the same rules, everything we prove about the rules applies to BOTH systems.

This is the most powerful abstraction in mathematics.

## 1. Groups — One Operation, Symmetry

A **group** (G, ∗) is a set G with an operation ∗ that satisfies four properties:

| Property | Meaning | Example (integers under +) |
|----------|---------|---------------------------|
| **Closure** | a ∗ b is always in G | Adding two integers gives an integer |
| **Associativity** | (a ∗ b) ∗ c = a ∗ (b ∗ c) | (1+2)+3 = 1+(2+3) |
| **Identity** | There exists e such that a ∗ e = e ∗ a = a | 0 is the identity for + |
| **Inverse** | For every a, there exists b such that a ∗ b = e | The inverse of 5 is -5 |

That's it. Four rules. An enormous amount of mathematics follows.

### Examples of groups

| Group | Set | Operation | Identity | Inverse of a |
|-------|-----|-----------|----------|-------------|
| (Z, +) | Integers | Addition | 0 | -a |
| (Q\{0}, ×) | Non-zero rationals | Multiplication | 1 | 1/a |
| (Z/nZ, +) | Integers mod n | Addition mod n | 0 | n - a |
| Symmetries of a square | 8 rotations/reflections | Composition | Do nothing | Undo the move |
| Permutations of {1,...,n} | All rearrangements | Composition | Identity permutation | Reverse the rearrangement |

### Commutativity (Abelian groups)
If a ∗ b = b ∗ a for all a, b, the group is **abelian** (commutative).
- (Z, +) is abelian: 3 + 5 = 5 + 3
- Permutations are NOT abelian: doing rotation then reflection ≠ reflection then rotation

### Key concepts

**Order:** The number of elements in a group. |Z/5Z| = 5. The group of symmetries of a square has order 8.

**Subgroup:** A subset that is itself a group under the same operation. Even integers form a subgroup of (Z, +).

**Lagrange's theorem:** The order of a subgroup divides the order of the group. If a group has 12 elements, its subgroups can only have 1, 2, 3, 4, 6, or 12 elements.

**Cyclic groups:** Generated by a single element. Z/nZ is cyclic — every element is some multiple of 1.

### Why groups matter

Groups capture **symmetry** — the most fundamental concept in physics. Every conservation law in physics (energy, momentum, charge) corresponds to a symmetry group (Noether's theorem). The Standard Model of particle physics is built on the group SU(3) × SU(2) × U(1).

In CS: permutation groups underlie sorting algorithms. Cyclic groups power modular arithmetic in cryptography.

## 2. Rings — Two Operations

A **ring** (R, +, ×) is a set with TWO operations that interact:

1. (R, +) is an abelian group (addition works, has identity 0, has inverses)
2. Multiplication is associative: (a × b) × c = a × (b × c)
3. Distributive law: a × (b + c) = a×b + a×c

Rings model systems where you can add, subtract, and multiply — but not necessarily divide.

### Examples

| Ring | What it is |
|------|-----------|
| (Z, +, ×) | Integers — the archetypal ring. Can't always divide. |
| Polynomials R[x] | Polynomials with real coefficients. Add and multiply them, but can't always divide evenly. |
| n×n matrices | Square matrices under matrix addition and multiplication. |
| Z/nZ (n not prime) | Integers mod n. Has zero divisors when n isn't prime: in Z/6Z, 2×3 = 0. |

### Zero divisors
In Z, if ab = 0, then a = 0 or b = 0. But in Z/6Z: 2 × 3 = 0, yet neither 2 nor 3 is 0. These are **zero divisors** — they break the ability to cancel.

An **integral domain** is a ring with no zero divisors. Z is an integral domain. Z/6Z is not.

## 3. Fields — Complete Arithmetic

A **field** (F, +, ×) is a ring where every non-zero element has a multiplicative inverse.

In a field, you can add, subtract, multiply, AND divide (by non-zero). Arithmetic is "complete."

### Examples

| Field | What it is |
|-------|-----------|
| Q | Rationals |
| R | Reals |
| C | Complex numbers |
| Z/pZ (p prime) | Integers mod a prime — the finite fields that power cryptography |
| GF(2) = {0, 1} | Binary field. + is XOR, × is AND. Foundation of digital circuits. |

### Why "p must be prime" for Z/pZ to be a field
If p is not prime, say p = 6, then 2 × 3 = 0, so 2 has no multiplicative inverse (it's a zero divisor). When p IS prime, Bézout's identity guarantees every non-zero element has an inverse.

### Finite fields (Galois fields)
For every prime p and positive integer n, there exists exactly one finite field with p^n elements, written GF(p^n).

- GF(2) powers binary arithmetic
- GF(2⁸) = GF(256) powers AES encryption (each byte is a field element)
- GF(p) for large primes powers RSA and Diffie-Hellman

## 4. Vector Spaces — Algebra Meets Geometry

A **vector space** over a field F is a set V with:
1. **Vector addition:** u + v (combining vectors)
2. **Scalar multiplication:** c · v (scaling vectors by field elements)

...satisfying natural axioms (associativity, distributivity, identity, etc.).

### Examples

| Vector space | Vectors | Scalars (field) |
|-------------|---------|-----------------|
| R² | Arrows in the plane | Real numbers |
| R³ | Arrows in 3D space | Real numbers |
| Rⁿ | n-tuples of reals | Real numbers |
| Polynomials of degree ≤ n | Polynomials | Real numbers |
| Continuous functions R → R | Functions | Real numbers |
| GF(2)ⁿ | Binary strings of length n | GF(2) |

The last one is critical — binary codewords in error-correcting codes are vectors over GF(2).

### Key concepts

**Basis:** A minimal set of vectors that can build every vector in the space through linear combinations. In R³, {(1,0,0), (0,1,0), (0,0,1)} is the standard basis.

**Dimension:** The number of vectors in a basis. R³ is 3-dimensional. The space of degree ≤ n polynomials is (n+1)-dimensional.

**Linear independence:** Vectors are independent if none can be written as a combination of the others. The basis must be independent.

**Subspace:** A subset that is itself a vector space. Lines and planes through the origin are subspaces of R³.

Vector spaces are the launching pad for [Linear Algebra](linear-algebra.md), which is the single most applied area of mathematics.

## 5. Homomorphisms — Structure-Preserving Maps

A **homomorphism** is a function between algebraic structures that preserves the operations.

**Group homomorphism** f: G → H satisfies f(a ∗ b) = f(a) ∗ f(b).
**Ring homomorphism** preserves both + and ×.

### Why this matters
If two structures are connected by a homomorphism, anything you prove about the structure of one transfers to the other. This is how mathematicians avoid reproving the same theorem in 50 different contexts.

**Isomorphism:** A bijective homomorphism. Two structures are isomorphic if they're "the same" in every structural way — they differ only in what we call the elements.

Example: (Z/2Z, +) and ({even, odd}, addition of parity) are isomorphic. They're the same group wearing different names.

**In CS:** Type isomorphisms in programming. `(A, B)` and `(B, A)` are isomorphic types. `A → (B → C)` and `(A, B) → C` are isomorphic (currying).

## 6. The Hierarchy

```
Set                     — just objects, no structure
  │
  └── Group             — one operation with identity and inverses
        │
        └── Ring        — two operations (add/multiply), but no guaranteed division
              │
              ├── Integral Domain — ring with no zero divisors
              │         │
              │         └── Field — every non-zero element has a multiplicative inverse
              │
              └── (Vector Space) — a ring (field) acting on another group
```

Each level adds more structure and more power. The tradeoff: more structure = more theorems, but fewer examples.

## What Algebra Powers

| Technology | Algebraic structure |
|------------|-------------------|
| **Cryptography (RSA)** | Arithmetic in Z/nZ (ring), Chinese Remainder Theorem |
| **AES encryption** | Arithmetic in GF(2⁸) (finite field) |
| **Error-correcting codes** | Linear algebra over finite fields |
| **Digital circuits** | Boolean algebra (ring Z/2Z) |
| **Computer graphics** | Transformation groups (rotation, scaling, translation) |
| **Quantum computing** | Unitary group U(n) acting on vector spaces |
| **Physics** | Symmetry groups dictate conservation laws and particle interactions |
| **Type systems** | Algebraic data types: sum types (OR), product types (AND) |

---

## Key Insight

Algebra is mathematics becoming aware of its own patterns. Instead of proving things about specific numbers or equations, it proves things about STRUCTURE — and those proofs apply everywhere that structure appears. When a cryptographer works in GF(2⁸) and a physicist works in SU(3), they're using the same theorems, because the underlying algebra is the same.
